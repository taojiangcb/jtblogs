3. 关键实现逻辑说明

  1. 大小判断：

    • 通过 Buffer.byteLength 获取文件实际大小
    • 通过 limit 参数控制阈值（默认 8KB）

  2. Base64 转换：

    • 使用 mime 包自动识别 MIME 类型
    • 生成标准的 Data URL 格式

  3. 大文件回退：

    • 直接调用 file-loader 处理大文件
    • 复用 Webpack 的资源哈希命名机制

  4. 路径处理：

    • 输出路径适配 Next.js 的静态资源结构
    • 文件名包含哈希值确保缓存有效性

    • 根据项目实际情况调整 limit 值
    • 建议对 WebP/AVIF 等现代格式使用更低阈值
    • 生产环境建议开启 CDN 路径配置

  3. 版本兼容性：

    • 适用于 Webpack 4/5
    • 兼容 Next.js 10+ 版本

替代方案对比

方案 优点 缺点
本文自定义 loader 完全控制转换逻辑 需要维护自定义代码
url-loader + file-loader 社区维护、开箱即用 配置灵活性较低
图片服务自动优化 无需本地处理 依赖外部服务

如果需要更复杂的处理（如自动生成 WebP 格式），建议结合 sharp-loader 或 image-webpack-loader 使用。
